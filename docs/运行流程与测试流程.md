# 围标串标 AI 服务测试

## 1. 启动 API 服务

```bash
python collusive_check_api.py
```

## 2. Java 后端接口测试（curl，与后端调用完全一致）

### 2.1 生成请求文件并启动文件服务


## 3. 本地开发测试（免 HTTP 文件服务）

## 4. 更换测试数据

PDF 文件名格式 `{recordId}_{bidderName}.pdf`，放入新目录：

```bash
python scripts/gen_test_payloads.py batch_downloads/新batchId
```

本地开发可用此命令直接运行：

```bash
python scripts/test.py --pdf-dir batch_downloads/新batchId
```

只换内容不换公司名：替换 PDF 后 `rm -rf collusive_workdir/batch_{batchId}/`

## 5. 结果

终端输出每个 itemCode 的 FAILED / SUCCESS / ERROR 和耗时。

- `status=ERROR` → 查 `collusive_workdir/batch_{batchId}/output/error_report.jsonl`
- OOM → `.env` 中 worker 数全部设为 1

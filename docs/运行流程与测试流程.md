# 围标串标 AI 服务测试流程

## 1. 支持的 itemCode

| itemCode | 名称 | 类型 | 是否重型 |
| :--- | :--- | :--- | :--- |
| `FILE_CODE_SIMILAR` | 文件码雷同 | PDF 指纹 | 否 |
| `EDITOR_SIGNER_SIMILAR` | 编辑经办人雷同 | PDF 元数据 | 否 |
| `DOC_AUTHOR_SIMILAR` | 文档作者雷同 | PDF 元数据 | 否 |
| `SAME_BID_CONTACT_SIMILAR` | 人名雷同 | 正文联系人提取 | 否 |
| `SAME_bidderName_SIMILAR` | 公司名雷同 | 正文公司名提取 | 否 |
| `TECH_BID_SIMILAR` | 技术标雷同 | 文本+图片+OCR | 是 |
| `Business_BID_SIMILAR` | 商务标雷同 | 文本+图片+OCR | 是 |

## 2. 启动服务

```bash
export OCR_ENGINE=rapidocr
export COLLUSIVE_ENABLE_OCR=1
export COLLUSIVE_ENABLE_IMAGE_ANALYSIS=1
export USE_GPU=0
export SBERT_DEVICE=cpu
export PHASE1_WORKERS=2
export PHASE3_WORKERS=2
export PDF_CHUNK_WORKERS=2
export OCR_COLLECT_WORKERS=2
export OCR_WORKERS=1

python collusive_check_api.py
```

## 3. 测试命令

测试脚本 `scripts/perf_test_collusive_api.py` 会临时启动 HTTP 文件服务，把本地 PDF 目录作为 URL 提供给 API。

### 3.1 轻量项测试（文件码/作者/经办人/人名/公司名）

```bash
python scripts/perf_test_collusive_api.py \
  --pdf-dir batch_downloads/75689 \
  --api http://127.0.0.1:8001 \
  --items FILE_CODE_SIMILAR DOC_AUTHOR_SIMILAR EDITOR_SIGNER_SIMILAR \
          SAME_BID_CONTACT_SIMILAR SAME_bidderName_SIMILAR
```

```bash
  --batch-id 10001 \
```

## 5. 测试结果解读

终端输出示例：

```text
[TECH_BID_SIMILAR] status=200 elapsed=xxx.xx s
results=2, failed=0, errors=0
```

- `status=200` — HTTP 请求成功
- `errors=0` — 无程序异常
- `failed=N` — N 家公司业务检测不通过
- `elapsed` — 当前 itemCode 总耗时

JSON 报告输出路径：

```text
collusive_workdir/batch_{batchId}_{dimension}/output/detection_report.json
```

## 6. 常见问题

| 问题 | 原因 | 处理 |
| :--- | :--- | :--- |
| `Connection refused` | 服务未启动 | `ps -ef \| grep collusive_check_api` 确认进程；`ss -lntp \| grep 8001` 确认端口 |
| `Killed` / `已杀死` | OOM | 降低 worker 数：`PHASE1_WORKERS=1 PDF_CHUNK_WORKERS=1 OCR_COLLECT_WORKERS=1 PHASE3_WORKERS=1` |
| `status=ERROR` | 程序异常 | 查看服务端日志堆栈 |
| RapidOCR 加载失败 | 模型未下载 | `python -c "from rapidocr_onnxruntime import RapidOCR; RapidOCR()"` |
